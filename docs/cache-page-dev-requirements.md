# Cache Page 开发需求说明

## 1. 背景

当前项目已经具备 Slurm 集群、作业、节点、资源和缓存基础能力，但距离“面向 Slurm 集群数据监控与分析的网站”还有一段差距。`Cache` 页面是设置中心里直接反映缓存健康度和查询效率的入口，用户反馈该页面右侧存在明显空白，影响可读性，也削弱了“资源利用率分析平台”应有的信息密度。

本轮开发先解决 `Cache` 页面空白问题，同时把后续面向资源利用率分析的网站能力整理成明确的扩展方向。

## 2. 问题描述

- `SettingsCacheStatistics` 使用固定双栏布局，右侧栏位默认用于环形图。
- 当缓存统计总量为 0，或图表没有有效内容时，右侧栏位仍然保留，视觉上表现为空白块。
- `SettingsCacheMetrics` 在“指标已开启但当前范围内无样本”时没有空态说明，用户只能看到空图区域。
- `SettingsCache` 在 `cluster.metrics = false` 时直接不显示时间序列内容，也没有解释说明，页面节奏不完整。

## 3. 明确需求

### 3.1 本次必须完成

- 修复 `Cache` 页面右侧空白问题。
- 统计面板在无缓存流量时必须显示明确的空态说明，不能出现无意义的白块。
- 图表开启但当前时间范围无数据时，必须显示可理解的空态说明。
- 集群未开启 metrics 时，页面必须解释为什么没有时间序列图。
- 保持当前前端视觉语言，不做大范围风格重构。

### 3.2 本次不做

- 不修改后端 cache 接口结构。
- 不新增新的 cache 统计字段。
- 不调整权限模型。

## 4. 实现方案

### 4.1 页面层

- `frontend/src/views/settings/SettingsCache.vue`
- 将 cache 统计和 metrics 组织为统一的纵向内容块。
- 当 cluster 未开启 metrics 时，补充说明卡片，解释“为什么没有实时趋势图”。

### 4.2 统计组件

- `frontend/src/components/settings/SettingsCacheStatistics.vue`
- 将统计区域改为“表格 + 概览面板”结构。
- 仅在有真实 hit/miss 总量时采用桌面双栏布局。
- 在无流量时自动退化为单列，避免桌面右侧保留空白栏。
- 概览面板增加：
  - 总请求数
  - 总命中率
  - key group 数量
- 环形图保留，但在无流量时隐藏，改为“等待缓存活动”的空态文案。
- 统计表按 hit/miss key 并集展示，避免 miss-only 或 hit-only key 丢失。

### 4.3 指标组件

- `frontend/src/components/settings/SettingsCacheMetrics.vue`
- 保留现有时间范围切换。
- 增加“当前范围无样本”的空态说明。
- 只有在存在有效样本时才显示直方图。

## 5. 影响范围

- 前端设置页 cache 相关渲染逻辑
- 前端 cache 页面测试
- 无后端接口改动
- 无数据库改动

## 6. 验收标准

- `Cache` 页面在缓存总量为 0 时，不再出现右侧空白块。
- `Cache` 页面在 metrics 开启但无样本时，展示明确说明。
- `Cache` 页面在 metrics 关闭时，展示明确说明。
- 统计表能显示 hit/miss key 并集。
- 相关前端单元测试通过。
- cache 相关后端测试回归通过。

## 7. 后续可扩展功能建议

为了把当前项目完善成 Slurm 集群数据监控分析网站，建议按下面方向逐步补强：

### 7.1 资源利用率总览

- 集群级 CPU、内存、GPU 利用率总览
- 分区级利用率排行
- 实时空闲资源与已分配资源趋势
- 超卖、热点分区、长期低利用率分区识别

### 7.2 作业效率分析

- 作业等待时间、运行时间、排队原因分布
- 大作业、长作业、失败作业趋势
- 用户和账号维度的资源占用分析
- 历史作业资源申请值与实际使用值对比

### 7.3 节点健康与异常定位

- 节点 CPU load、内存、磁盘、网络健康度
- drain、down、flapping 节点追踪
- GPU 节点温度、显存、利用率监控
- 异常节点时间线和告警闭环

### 7.4 容量规划与预测

- 过去 7/30/90 天资源占用趋势
- 高峰时段识别
- 分区容量缺口预测
- 扩容建议和历史对比

### 7.5 告警与通知

- 资源利用率阈值告警
- 分区拥塞告警
- 节点异常告警
- cache、agent、slurmrestd 连通性告警

### 7.6 多维分析能力

- 用户、账号、QOS、分区、节点类型多维交叉筛选
- TopN 用户、账号资源占用排行
- 作业模板、工具链、项目组资源画像
- 导出报表和管理周报
