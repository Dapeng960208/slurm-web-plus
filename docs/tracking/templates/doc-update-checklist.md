# 文档更新检查清单

在结束开发前，逐项确认：

- [ ] 新功能是否已经写入 `docs/tracking/`
- [ ] 新路由/页面是否写入正式文档
- [ ] 新接口是否写入正式文档
- [ ] 新配置项是否写入正式文档
- [ ] 新权限动作是否写入正式文档
- [ ] 新数据库迁移是否写入正式文档
- [ ] 功能开启条件和降级行为是否写清楚
- [ ] 测试或验证入口是否补充
- [ ] `docs/README.md` 是否已补入口
- [ ] 文件命名与放置是否符合 `docs/standards/document-naming-convention.md`
- [ ] 开发过程中遇到的可复现错误是否已记录到 `docs/tracking/error-log.md`
- [ ] 提交前已检查 `git status --porcelain`，并确认未提交改动是否应纳入本次提交
- [ ] 文档内容是否与当前代码一致
